# Repository Guidelines for AI Agents

This file is read by automated agents (including Repo Assist) before performing any work on this repository.

## Project Overview

`@eresearchqut/ddb-repository` is a TypeScript library providing a generic repository pattern implementation for AWS DynamoDB, simplifying CRUD operations and common database interactions. It is published publicly to npm.

## Tech Stack

- **Language**: TypeScript (strict)
- **Runtime target**: Node.js (LTS)
- **Build tool**: `tsdown` (produces both ESM and CJS output in `dist/`)
- **Test framework**: Vitest (with `@vitest/coverage-v8` for coverage)
- **Linter**: oxlint
- **Release**: `semantic-release` (conventional commits drive versioning and CHANGELOG)
- **AWS SDK**: `@aws-sdk/client-dynamodb` v3 + `@aws-sdk/util-dynamodb`

## Project Structure

```
src/                    # Source files (TypeScript)
  DynamoDbRepository.ts # Main repository class and types
  consumed-capacity-middleware.ts
  index.ts              # Public API exports
test/                   # Vitest tests (mirrors src/ structure)
dist/                   # Build output (do not edit; generated by `tsdown`)
.github/workflows/
  build.yml             # CI: lint, test (with coverage), build
  release.yml           # CD: semantic-release on push to main
```

## Commands

```bash
npm run build          # Compile TypeScript → dist/
npm run test           # Run Vitest tests
npm run test:coverage  # Run tests with coverage report
npm run lint           # oxlint check
npm run lint:fix       # oxlint auto-fix
```

Tests use `testcontainers` to run DynamoDB Local for integration tests — Docker must be available.

## Coding Conventions

- **No comments** unless the code genuinely needs clarification — keep code self-documenting.
- **TypeScript strict mode** — no implicit `any`, no unnecessary type assertions.
- Prefer `async/await` and method chaining over callbacks.
- All public API types and the main class are exported from `src/index.ts`.
- Use `marshall`/`unmarshall` from `@aws-sdk/util-dynamodb` for all DynamoDB attribute conversions.
- `removeUndefinedValues: true` should be passed to `marshall` to avoid DynamoDB errors.

## Contribution Rules

- **No breaking changes** to the public API without explicit discussion in an issue first.
- **No new runtime dependencies** without opening an issue and getting maintainer approval — this is a library with peer dependencies.
- **Small, focused PRs** — one concern per PR.
- All PRs must pass CI (lint + tests + build) before merging.
- Commits must follow [Conventional Commits](https://www.conventionalcommits.org/) (`feat:`, `fix:`, `chore:`, `docs:`, etc.) — semantic-release uses these to determine the version bump and CHANGELOG entry.
- PRs target the `main` branch (releases) or `develop` (integration).

## Release Process

Releases are fully automated via `semantic-release` on push to `main`. Do not manually bump versions in `package.json`. A `feat:` commit triggers a minor bump; `fix:` triggers a patch; `feat!:` or `BREAKING CHANGE:` triggers a major bump.
